# Intelligent Complaint Analysis Chatbot for CrediTrust Financial

This repository contains the complete source code, documentation, and setup guides for an AI-powered internal chatbot that analyzes and summarizes customer complaints. The tool transforms large volumes of unstructured complaint data into actionable insights, enabling business teams to identify trends, understand customer pain points, and make data-driven decisions efficiently.

---



## 🎯 Project Overview

CrediTrust Financial, a rapidly growing digital finance company, receives thousands of customer complaints monthly across its various channels. Currently, internal teams like product management manually sift through this raw data, a process that is time-consuming and inefficient.

This project's mission is to build an intelligent, internal tool that transforms this unstructured complaint data into a strategic asset. We are developing a Retrieval-Augmented Generation (RAG) powered chatbot that enables non-technical users, such as a Product Manager, to ask plain-English questions about customer issues and receive synthesized, evidence-backed answers in seconds.

---

## ✨ Key Features

- **Semantic Search**: Utilizes vector embeddings to find complaint narratives that are most relevant to a user's query, going beyond simple keyword matching.
- **Insightful Answer Generation**: Leverages a Large Language Model (LLM) to read the retrieved complaints and generate a concise, human-readable summary that directly answers the user's question.
- **Multi-Product Analysis**: Supports filtering and comparison of issues across all of CrediTrust's major financial services.
- **Trust and Verification**: The final interface will display the source text chunks used by the LLM to generate an answer, allowing for immediate verification and building user trust.

## 🛠️ Technology Stack

- **Data Analysis**: Pandas, Seaborn, Matplotlib
- **Core RAG Pipeline**: LangChain
- **Embedding Model**: SentenceTransformers (all-MiniLM-L6-v2)
- **Vector Database**: ChromaDB or FAISS
- **Web Interface**: Gradio or Streamlit
- **Environment**: Python, Jupyter Notebooks

## ⚙️ Setup and Installation

### Prerequisites

- Ensure you have **Git** installed.
- Ensure you have **Python 3.8 or higher** installed.

### Clone the Repository

Open your terminal, navigate to where you want to store the project, and run:

```bash
git clone <your-repository-url>
cd credittrust_rag_chatbot
```

### Create and Activate a Virtual Environment
A virtual environment is critical for managing project-specific dependencies.

# Create the virtual environment
```bash
python3 -m venv venv

# Activate it on macOS/Linux
source venv/bin/activate

# Or activate it on Windows
.\venv\Scripts\activate
```

### Install Dependencies
Install all the required packages using the requirements.txt file.
```bash
pip install -r requirements.txt
```


# 🚀 Task Execution and Deliverables

#### ✅ 1. Placing Raw Data

- **Task**: I downloaded the CFPB complaint dataset and placed it inside the `data/` directory as `complaints.csv`.
- **Coverage**: This dataset included thousands of real-world financial complaints, which formed the foundation of the entire project pipeline.

---

#### ✅ 2. Performing EDA and Preprocessing

- **Task**: I explored the dataset using a Jupyter Notebook (`notebooks/01_Eda.ipynb`).
- **Coverage**:
  - Analyzed distributions and key trends within the complaints data.
  - Filtered complaints to focus on relevant financial products aligned with CrediTrust’s portfolio.
  - Cleaned and preprocessed the complaint narratives to remove noise and prepare them for embedding.
- **Outcome**: Generated a cleaned dataset `filtered_complaints.csv` saved in the `data/` directory for further processing.

---

#### ✅ 3. Building the Vector Store

- **Task**: Using the script `src/build_vector_store.py`, I transformed the cleaned complaints into a semantic search database.
- **Coverage**:
  - **Chunked** the text narratives for manageable and meaningful embedding.
  - Generated **vector embeddings** for each chunk using the SentenceTransformers model.
  - Indexed these embeddings into **ChromaDB / FAISS**, creating a persistent and efficient vector store.
- **Outcome**: A set of vector database files were created in the `vector_store/` directory, enabling fast semantic search queries for the chatbot.
